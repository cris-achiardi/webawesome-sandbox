# Web Awesome — Component Architecture Audit & Contracts

> Hand-authored from a source read of `@awesome.me/webawesome@3.9.0` (`dist/chunks/*.js`,
> `dist/custom-elements.json`, `dist/skills/`). Unlike [`prop-map.md`](./prop-map.md) and
> [`tokens.md`](./tokens.md), this file is **not** regenerated by `npm run docs` — update it by hand
> after a major WA version bump.
>
> **Purpose:** document the dev patterns every WA component follows, then turn them into **contracts**
> so any custom component we add to the design system has the same architecture, lifecycle, and public
> surface as a first-party WA component. Pair with [`naming-conventions.md`](./naming-conventions.md)
> for the naming rules these contracts assume.

---

## 0. TL;DR — the shape of a WA component

Every one of the 65 components is a **Lit element** that:

1. extends one of **two** base classes — `WebAwesomeElement` (51) or `WebAwesomeFormAssociatedElement` (14);
2. is registered with the `@customElement('wa-*')` decorator and a `wa-` tag;
3. declares its public API with Lit `@property()` decorators (camelCase field ↔ kebab-case attribute);
4. exposes presentation via **CSS parts** (`part="base"`), **slots**, **CSS custom properties**, and
   **custom states** (`:state(...)`) — never via leaked internal classes;
5. layers styles as `static css = [shared_styles, component_styles]`, with host styles auto-prepended;
6. emits **custom events** prefixed `wa-` (and re-emits a curated set of native events);
7. reacts to attribute changes through a `@watch()`-decorated `handleXChange()` method.

If a custom component honors §7 (Contracts) below, it is indistinguishable from a WA component to a
consumer, a theme, or a Figma integration.

## 1. Class hierarchy

```
LitElement
└── WebAwesomeElement                    (base for all 65; 51 use it directly)
    └── WebAwesomeFormAssociatedElement  (14 form controls)
```

### 1.1 `WebAwesomeElement` (`src/internal/webawesome-element.ts`)

The universal base. Every component gets, for free:

| Feature | How it works |
| --- | --- |
| **Host reset styles** | `static get styles()` prepends `host_styles_default` (box-sizing, `[hidden]` handling) to each component's `css`. |
| **Custom-state helper** | `this.customStates.set(name, bool)` / `.has(name)` wraps `ElementInternals.states` with browser-support guards. Sets `wa-defined` on construct. |
| **`ElementInternals`** | Attached in the constructor (`this.internals = this.attachInternals()`). |
| **SSR awareness** | `this.didSSR` flag; hydration-error event dispatch in `update()`; reflected as `did-ssr` attribute. |
| **Reflected-prop restoration** | Records initial reflected props and restores them if an attribute is later removed (`willUpdate`). |
| **`dir` / `lang`** | Inherited reactive properties on the base. |
| **`setStyle` / `setStyleProperty`** | SSR-safe ways to write inline styles / CSS custom props. |
| **Docs comment** | Injects a `<!-- Web Awesome: https://webawesome.com/docs/components/<name> -->` comment into the shadow root. |

### 1.2 `WebAwesomeFormAssociatedElement` (`src/internal/webawesome-form-associated-element.ts`)

Adds the **Constraint Validation** layer for form controls. Sets `static formAssociated = true`.

| Feature | How it works |
| --- | --- |
| **Shared form props** | `name` (reflected), `disabled`, `required`. |
| **Validator pipeline** | `static get validators()` returns an array of validator objects (`{ observedAttributes, checkValidity(el) }`). A component appends its own with `[...super.validators, MirrorValidator()]`. `CustomErrorValidator` is always present. |
| **CVAPI surface** | `checkValidity()`, `reportValidity()`, `setCustomValidity()`, `setValidity()`, `validity`, `validationMessage`, `willValidate`, `validationTarget`. |
| **Form lifecycle hooks** | `formResetCallback`, `formDisabledCallback`, `formStateRestoreCallback`. |
| **Form value sync** | `updateFormValue()` / `setValue()` push to `ElementInternals.setFormValue`; arrays become repeated `FormData` entries. |
| **Interaction tracking** | `assumeInteractionOn = ['input']` (overridable, e.g. input uses `['blur','input']`); flips `hasInteracted`, which drives `:state(user-invalid)` / `:state(user-valid)`. |
| **Validity custom states** | `required`, `optional`, `valid`, `invalid`, `user-valid`, `user-invalid` all set automatically. |

**Rule:** a new form control extends `WebAwesomeFormAssociatedElement` and never re-implements
validation — it only adds validators and a `validationTarget` getter.

## 2. Registration & file layout

Observed for every component (e.g. `button`, `card`, `input`):

```
dist/components/<name>/
  <name>.js            ← re-exports the class (default + named) from its chunk
  <name>.d.ts
  <name>.styles.js     ← re-exports the component's compiled css
  <name>.styles.d.ts
```

- **Tag:** always `wa-<kebab-name>`; multi-word and `*-item` / `*-group` children follow suit
  (`wa-tab-group`, `wa-accordion-item`, `wa-number-input`). 100% of tags start with `wa-`.
- **Class name:** PascalCase `Wa<Name>` (`WaButton`, `WaTabGroup`).
- **Registration:** `WaButton = __decorateClass([customElement('wa-button')], WaButton)` — i.e. the
  TS `@customElement('wa-button')` decorator. Source files live at `src/components/<name>/<name>.ts`.
- **One element per file**, registered as a side effect of import. Storybook's generated
  `register-all.ts` relies on this (importing the module registers the element).
- After registration each class calls `disableWarning?.('change-in-update')`.

## 3. Property / attribute pattern

Properties are declared with `__decorateClass([property(...)], proto, name)` (the compiled form of
Lit's `@property`). The conventions, verified across all components:

| Intent | Decorator | Notes |
| --- | --- | --- |
| Enum axis (variant, size, appearance, orientation, placement…) | `@property({ reflect: true })` | Reflected so CSS can match `:host([size='m'])`. Default set in constructor. |
| Boolean flag | `@property({ type: Boolean, reflect: true })` | Reflected; opt-in flags use `with-*`, opt-out flags use `without-*` (see naming doc). |
| String value | `@property()` | e.g. `label`, `placeholder`, `href`. |
| camelCase → kebab attribute | `@property({ attribute: 'with-caret' })` | **103 props** use an explicit kebab `attribute`. Field is `withCaret`, attribute is `with-caret`. |
| Native HTML mirror | `@property({ attribute: 'formaction' })` | Mirrors of native attrs keep the native (all-lowercase, no dash) spelling: `formaction`, `formnovalidate`, `tabindex`. |
| Internal reactive state | `@state()` | Not an attribute; e.g. `invalid`, `isIconButton`. |
| Query a shadow node | `@query('.button')` | Caches a shadow-DOM reference (`this.button`). |

**Defaults live in the constructor**, not the decorator (`this.variant = 'neutral'`). Enum defaults
are also the reflected attribute's resting value.

### 3.1 Controlled `value` getter/setter (form controls)

Form controls don't expose `value` as a plain field. They use a private `_value` + `defaultValue`
and a getter/setter that tracks `valueHasChanged` (see `WaInput`). This separates the *default* (from
the `value` attribute at parse time) from the *current* value, so `formResetCallback` works.

## 4. Reactive side-effects: the `@watch` + `handleXChange` pattern

WA wraps Lit's `updated`/`willUpdate` in a `watch(propName, opts?)` decorator. The handler is **always**
named `handle<Prop>Change` (111 public methods mined; the dominant family is `handle*Change`).

```js
// from button.ts
handleDisabledChange() {
  this.customStates.set('disabled', this.disabled);
  this.updateValidity();
}
// decorated:
__decorateClass([ watch('disabled', { waitUntilFirstUpdate: true }) ],
  WaButton.prototype, 'handleDisabledChange', 1);
```

- `watch('size')` → fires on every change including first render.
- `watch('disabled', { waitUntilFirstUpdate: true })` → skips the initial render.
- Slot-driven recalculation goes in `willUpdate()` (see card recomputing `withHeader` etc.).

**Rule:** any prop with a visual/behavioral side-effect gets a `watch()` + `handle<Prop>Change`.
Don't put side-effects inline in setters.

## 5. Rendering & presentation surface

### 5.1 `render()` conventions (from button/card/input)

- Returns a Lit `html\`...\`` template (static-html `literal` when the tag itself is dynamic — e.g.
  button renders `<a>` vs `<button>` based on `href`).
- The outermost rendered element is `part="base"`.
- Class names on internal nodes are driven by `classMap({...})`; conditional attributes by `ifDefined`.
- Every slot that has styleable structure also carries a `part`:
  `<slot name="start" part="start" class="start">`.

### 5.2 The four presentation channels (the public styling contract)

A consumer must be able to restyle a component **without** reaching into its shadow DOM internals.
WA exposes exactly four channels — a custom component must expose the same:

| Channel | Mechanism | Count in WA | Example |
| --- | --- | --- | --- |
| **CSS Parts** | `part="..."` on internal nodes | 171 parts | `wa-button::part(base)` |
| **Slots** | named `<slot>` | 52 slots | `<wa-input><span slot="hint">` |
| **CSS Custom Properties** | component-scoped `--name` (no `wa-` prefix) | 93 props | `wa-dialog { --width: 30rem }` |
| **Custom States** | `internals.states` → `:state(name)` | 19 states | `wa-checkbox:state(checked)` |

> Note the prefix split: **global design tokens are `--wa-*`** (see `tokens.md`); **per-component
> customization props are unprefixed** (`--width`, `--size`, `--track-color`). Don't mix them up.

### 5.3 Style composition

```js
WaButton.css  = [button_styles_default, variants_styles_default, size_styles_default];
WaCard.css    = [size_styles_default, card_styles_default];
WaInput.css   = [size_styles_default, form_control_styles_default, input_styles_default];
```

- `static css` is an **array** of `css\`\`` tagged templates; `WebAwesomeElement.styles` prepends the
  shared host reset automatically.
- **Shared style modules** are reused across components: `size.styles` (the `:host([size=...])`
  font-size scale, with `s/small`, `m/medium`, `l/large` aliases), `variants.styles`,
  `form-control.styles`. A new component reuses these rather than re-deriving the size scale.
- Components style themselves through **`--wa-*` tokens only** — never hard-coded colors/spacing.

## 6. Events

Two tiers, both observed in the data (37 distinct events):

1. **WA custom events — always `wa-` prefixed.** Lifecycle verbs are paired:
   `wa-show`/`wa-after-show`, `wa-hide`/`wa-after-hide`, `wa-expand`/`wa-collapse`,
   `wa-load`/`wa-error`. State events: `wa-change`-style intents use `wa-select`,
   `wa-selection-change`, `wa-clear`, `wa-invalid`, `wa-reposition`.
   Emitted via a typed event class (`new WaInvalidEvent()`, `new WaClearEvent()`).
2. **Re-emitted native events — NOT prefixed:** `blur`, `focus`, `change`, `input`, `beforeinput`,
   `load`, `error`. Form controls re-dispatch the native event from the host using
   `relayNativeEvent(event, { bubbles: true, composed: true })` so framework `(change)` / `@change`
   bindings work natively.

**Rule:** anything WA-specific → `wa-<verb>` typed event; anything that mirrors a native control →
re-emit the native event name. Provide `wa-after-*` companions for any animated show/hide.

## 7. Contracts — checklist for a new custom component

Use this as the definition-of-done for any component added alongside WA. Each item maps to a section
above and a rule in [`naming-conventions.md`](./naming-conventions.md).

### 7.1 Structure
- [ ] Tag is `wa-<kebab>` *(or your DS prefix — see naming doc §1; pick one and never mix)*.
- [ ] Class is `PascalCase` matching the tag; one element per file at `components/<name>/<name>.ts`.
- [ ] Extends `WebAwesomeElement`, or `WebAwesomeFormAssociatedElement` if it submits a form value.
- [ ] Registered with `@customElement('<tag>')`; styles in a sibling `<name>.styles.ts`.

### 7.2 API surface
- [ ] Enum axes reuse the shared vocabulary (`variant`, `appearance`, `size`, `orientation`,
      `placement`) with the **same value sets** as WA — don't invent a parallel size scale.
- [ ] Props are camelCase fields; multi-word attributes are kebab-case via `attribute:`.
- [ ] Enum + boolean props that affect styling are `reflect: true`.
- [ ] Opt-in features named `with-*`; opt-out named `without-*`.
- [ ] Defaults set in the constructor.
- [ ] Every side-effecting prop has a `@watch()` + `handle<Prop>Change`.

### 7.3 Presentation
- [ ] Outer node is `part="base"`; every styleable internal node has a `part`.
- [ ] All customization goes through parts / slots / unprefixed `--custom-props` / `:state()` —
      no consumer should need to pierce the shadow root.
- [ ] Styles use `--wa-*` tokens exclusively (no literal colors, sizes, radii).
- [ ] Reuses shared style modules (`size.styles`, `form-control.styles`) where applicable.

### 7.4 Behavior
- [ ] WA-specific events are `wa-<verb>` typed `Event` subclasses; native-mirroring events re-emitted
      with their native names.
- [ ] Animated open/close exposes `wa-show`/`wa-after-show` (and hide counterparts).
- [ ] Form controls add validators via `static get validators()`, set `validationTarget`, and rely on
      the base CVAPI — they do not hand-roll validity.
- [ ] Custom states set through `this.customStates.set(...)` (never ad-hoc classes on the host).
- [ ] Public imperative methods follow WA names (`focus`, `blur`, `show`, `hide`, `select`,
      `checkValidity`, `reportValidity`).
- [ ] SSR-safe: no `document`/`window` access outside guarded lifecycle; honor `this.didSSR`.

### 7.5 Accessibility
- [ ] Interactive icon-only controls require a `label` (button warns when an icon button has none).
- [ ] ARIA wiring lives in `render()` via `ifDefined`; focus is delegated
      (`shadowRootOptions.delegatesFocus = true`) where the host stands in for a control.

## 9. Deviations & gotchas found in the audit

- **Size aliases:** the scale is `xs · s · m · l · xl`, but `small`/`medium`/`large` are accepted
  aliases and `warnDeprecatedSize()` fires for deprecated spellings. Standardize on `xs…xl` in new
  components and Figma; treat the long forms as legacy.
- **`appearance` is overloaded:** its value union spans `accent · filled · outlined ·
  filled-outlined · plain` **plus** `button`/`default` on a couple of components. Constrain new
  components to the five-value paint set.
- **Native-attribute mirrors break the kebab rule on purpose:** `formaction`, `formnovalidate`,
  `tabindex`, `time-zone` mirrors keep native casing. This is intentional fidelity, not a bug —
  preserve it for any native mirror.
- **Very few deprecations:** only `wa-qr-code.fill` / `.background` are flagged deprecated in 3.9.0,
  so the surface is stable to build contracts against.
- **BEM-ish nested parts:** sub-component parts use `__` (e.g. `checkbox__base`,
  `tag__remove-button__base`). Follow this when a component embeds another WA component.
