# Web Awesome — Naming Conventions

> Hand-authored from a source read of `@awesome.me/webawesome@3.9.0`. Backed by the naming data mined
> from `dist/custom-elements.json` (65 components · 37 events · 171 parts · 52 slots · 93 component
> CSS props · 19 custom states · 103 camel→kebab attribute mappings · 111 public methods). Not
> regenerated by `npm run docs`.
>
> **Purpose:** a single normative table of *how everything is named* in WA, so (a) custom components
> match the library exactly and (b) a future **Figma integration** can enforce the same names on
> layers, components, variants, and tokens. Pair with [`component-architecture.md`](./component-architecture.md).

---

## 1. The master table

Every identifier kind a component exposes, its casing rule, the pattern, and verified examples.

| Identifier | Case | Pattern / rule | Examples (from WA) |
| --- | --- | --- | --- |
| **Custom element tag** | kebab-case | `wa-<noun>`; child parts `<parent>-item` / `<parent>-group` | `wa-button`, `wa-tab-group`, `wa-accordion-item`, `wa-number-input` |
| **Class name** | PascalCase | `Wa<TagInPascal>` | `WaButton`, `WaTabGroup`, `WaNumberInput` |
| **JS property** | camelCase | the canonical API name | `withCaret`, `playbackRate`, `mobileBreakpoint` |
| **HTML attribute** | kebab-case | camel property auto-mapped via `attribute:` | `with-caret`, `playback-rate`, `mobile-breakpoint` |
| ↳ *native-mirror attribute* | lowercase, **no dash** | mirrors of native HTML attrs keep native spelling | `formaction`, `formnovalidate`, `tabindex` |
| **Enum / axis value** | kebab-case | short, lowercase tokens | `neutral`, `filled-outlined`, `bottom-start`, `xs` |
| **Boolean opt-in flag** | kebab-case | `with-<feature>` | `with-header`, `with-clear`, `with-caret`, `with-markers` |
| **Boolean opt-out flag** | kebab-case | `without-<feature>` | `without-controls`, `without-spin-buttons`, `without-scrollbar` |
| **Custom event** | kebab-case | `wa-<verb>`; paired with `wa-after-<verb>` for "done" | `wa-show` / `wa-after-show`, `wa-select`, `wa-invalid` |
| ↳ *native re-emitted event* | lowercase, no prefix | keep the native name | `change`, `input`, `focus`, `blur`, `load`, `error` |
| **CSS Part** | kebab-case | semantic role; outer = `base`; nested = `parent__child` (BEM) | `base`, `label`, `clear-button`, `checkbox__control`, `tag__remove-button__base` |
| **Slot** | kebab-case | role-named; default slot is unnamed | `label`, `hint`, `start`, `end`, `header`, `footer`, `media`, `*-icon` |
| **CSS custom property** (component) | kebab-case | `--<name>`, **no `wa-` prefix** | `--width`, `--size`, `--track-color`, `--show-duration` |
| **Design token** (global) | kebab-case | `--wa-<category>-<...>` | `--wa-color-brand-fill`, `--wa-space-m`, `--wa-font-size-l` |
| **Custom state** | kebab-case | adjective/participle; queried via `:state(...)` | `checked`, `disabled`, `loading`, `user-invalid`, `icon-button` |
| **Change handler** | camelCase | `handle<Prop>Change` | `handleDisabledChange`, `handleValueChange`, `handleSizeChange` |
| **Imperative method** | camelCase | native-aligned verbs | `focus`, `blur`, `show`, `hide`, `select`, `checkValidity` |
| **Private field** | camelCase `_x` | leading underscore | `_value`, `_hasRecordedInitialProperties` |

## 2. Shared axis vocabulary (enforce these value sets)

These enum axes recur across many components. New components — and Figma variant properties — must
use the **same property names and value sets**, not synonyms. (Counts = components using the axis.)

| Axis (Figma variant prop) | Allowed values | Used on | Notes |
| --- | --- | --- | --- |
| `size` | `xs` · `s` · `m` · `l` · `xl` | 19 | Canonical scale. `small`/`medium`/`large` are **legacy aliases** — do not introduce in new work. |
| `appearance` | `accent` · `filled` · `filled-outlined` · `outlined` · `plain` | 14 | Visual fill (paint). Constrain to these five. |
| `variant` | `neutral` · `brand` · `success` · `warning` · `danger` | 6 | Semantic intent/color. `neutral` is the default. |
| `orientation` | `horizontal` · `vertical` | 9 | Layout axis. |
| `placement` | `top` · `bottom` · `left` · `right` (+ `-start` / `-end` variants) | 9 | Overlay anchor. 14 combinations total. |

**Value semantics** (for Figma tooltips / token descriptions):

| Value | Meaning |
| --- | --- |
| `neutral` | Default, low-emphasis intent. |
| `brand` | Primary brand color. |
| `success` / `warning` / `danger` | Positive / caution / destructive status. |
| `accent` | Solid brand-colored fill (highest emphasis). |
| `filled` | Tinted/neutral solid fill. |
| `outlined` | Border only, transparent fill. |
| `filled-outlined` | Filled with a visible border. |
| `plain` | No chrome until interaction. |

## 3. Slot naming vocabulary

Reuse these role names rather than coining new ones (52 slots total; the recurring families):

| Slot family | Names | Used for |
| --- | --- | --- |
| Edges | `start`, `end` | Leading/trailing adornments (icons, buttons). Prefer over `left`/`right` (RTL-safe). |
| Form metadata | `label`, `hint` | Field label and help text. |
| Container regions | `header`, `footer`, `media`, `actions`, `header-actions`, `footer-actions` | Card/panel structure. |
| Icon slots | `*-icon` | `clear-icon`, `collapse-icon`, `expand-icon`, `show-password-icon`, `decrement-icon`… |
| Page regions | `banner`, `navigation`, `navigation-header`, `navigation-footer`, `subheader`, `main-header`, `main-footer`, `aside`, `menu` | `<wa-page>` layout (see architecture doc / SKILL). |
| Default | *(unnamed)* | Primary content. |

## 4. Custom-state vocabulary

All 19 states, for `:state()` styling and Figma "state" layers. Boolean unless noted.

`animating` · `blank` · `checked` · `current` · `disabled` · `dragging` · `error` · `expanded` ·
`focused` · `hover` · `icon-button` · `indeterminate` · `link` · `loading` · `open` · `selected` ·
`success` · `user-invalid` · `user-valid`

> `user-invalid` / `user-valid` only activate after interaction (`hasInteracted`) — distinct from the
> always-current `invalid` / `valid`. Mirror this distinction in Figma validation states.

## 5. Token naming (global `--wa-*`)

Full values are generated in [`tokens.md`](./tokens.md). The naming grammar for enforcement:

```
--wa-<category>[-<subject>][-<variant>][-<scale>]
```

| Category | Shape | Example |
| --- | --- | --- |
| Color | `--wa-color-<role>-<variant>` | `--wa-color-brand-fill`, `--wa-color-surface-default` |
| Space | `--wa-space-<scale>` | `--wa-space-xs … --wa-space-3xl` |
| Font | `--wa-font-<facet>-<scale>` | `--wa-font-size-m`, `--wa-font-weight-bold` |
| Border | `--wa-border-<facet>-<scale>` | `--wa-border-radius-m`, `--wa-border-width-s` |
| Shadow | `--wa-shadow-<scale>` | `--wa-shadow-s … --wa-shadow-l` |
| Focus | `--wa-focus-ring-*` | `--wa-focus-ring-color` |
| Transition | `--wa-transition-<speed>` | `--wa-transition-fast` |

**Scales use the same `xs · s · m · l · xl` ladder as the `size` axis** (extended with `2xs`, `2xl`,
`3xl` where needed). Keep Figma token names 1:1 with these CSS variable names — same casing, same
scale words.

## 6. Figma integration rules

A direct mapping from WA's code conventions to Figma structure, so design and code stay in lockstep.

| Figma concept | Bind to WA… | Naming rule |
| --- | --- | --- |
| **Component / Component Set name** | element tag | Name the set after the tag minus prefix in PascalCase (`Button`, `TabGroup`), and record the tag (`wa-button`) in the description. |
| **Variant property** | reflected enum attribute | Property name = axis name (`size`, `variant`, `appearance`, `orientation`, `placement`). |
| **Variant property values** | enum value set | Exact WA values, kebab-case (`filled-outlined`, `bottom-start`, `xs`). No synonyms, no Title Case. |
| **Boolean variant** | boolean attribute | Use the attribute name verbatim (`with-header`, `without-controls`). |
| **Layer named for a slot** | named slot | Match the slot name (`start`, `hint`, `header-actions`). Default content layer = the unnamed slot. |
| **State (interaction) layer** | custom state | Match the `:state()` name (`hover`, `focused`, `checked`, `user-invalid`). |
| **Style / token reference** | `--wa-*` token | Figma variable name = CSS variable name, 1:1 (`color/brand/fill` ↔ `--wa-color-brand-fill`). |
| **Component property (text)** | string `@property` | camelCase property exposed as a Figma text/instance-swap property (`label`, `placeholder`). |

### Enforcement checklist (lint rules for the Figma sync)
- [ ] Reject variant property names not in the axis registry (§2) unless explicitly component-local.
- [ ] Reject enum values outside the documented set for a known axis.
- [ ] Reject Title Case / spaces in variant values, slot layers, or token names — kebab-case only.
- [ ] Boolean variants must be named `with-*` / `without-*`.
- [ ] State layers must be one of the 19 custom states (§4).
- [ ] Token/variable names must match an existing `--wa-*` token from `tokens.md`.
- [ ] Component set name's recorded tag must match `wa-<kebab>` (or the chosen DS prefix).

## 7. Quick "do / don't"

| ✅ Do | ❌ Don't |
| --- | --- |
| `size="s"` | `size="small"` (legacy alias) |
| `variant="danger"` | `variant="error"` / `variant="red"` |
| `with-header` boolean | `show-header` / `header-enabled` |
| `wa-after-show` event | `shown` / `onShow` / `afterShow` |
| `::part(base)` for the root | a leaked `.button` class |
| `--width` (component prop) | `--wa-width` (that prefix is for global tokens) |
| `--wa-space-m` token | a hard-coded `12px` |
| native `change` re-emitted | a custom `wa-change` that shadows native |
| RTL-safe `start` / `end` slots | `left` / `right` slots |
